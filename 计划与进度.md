# 理科实验室改造计划与进度

## 当前目标

将原 3D 元素周期表项目改造为 `science-lab`：保留现有元素周期表与化合物 3D 展示能力，并通过离线导入 + 原生渲染方式接入 Edulab 三类技能。

## 浏览支持边界

- 正式支持：桌面、横屏平板、展开后的折叠屏横屏。
- 最低支持视口：`820px × 560px`。
- 不支持：直板手机竖屏、平板竖屏、过窄分屏窗口。
- 低于支持范围时显示全屏提示，不提供滚动降级或手机竖屏适配。

  1. **KaTeX 导入路径错误**：`katexUtils.js` 之前从 `katex/contrib/auto-render` 导入，该路径在 katex 0.16.11 包中不存在（实际路径是 `katex/dist/contrib/auto-render.mjs`），导致模块加载失败，`renderLatex` 调用了但从未真正生效，公式一直以源码显示。修正为 `import renderMathInElement from 'katex/dist/contrib/auto-render.mjs'`。
  2. **解析几何动画引擎**（此前"已知问题"清单中的遗留任务，本次完整实现）：新增 `src/edulab/renderers/geometryEngine.js`，实现：
     - `solveDerived(board, paramValue)`：按 `board.derived` 数组顺序求解全部8种类型——`line_through_angle`（过定点定角度直线）、`line_through_points`（两点确定直线）、`intersect_line_conic`（直线与椭圆/双曲线/抛物线求交，各自实现了二次方程判别式求解）、`vector`、`segment`、`polygon`、`point_on_conic`（圆锥曲线参数方程取点）、`point_reflect`（中心对称点反射）。
     - `computeReadout(readout, solved, board, paramValue)`：补全全部9种缺失类型——`coord`/`slope`/`length`/`distance`/`area_triangle`（三角形面积叉积公式）/`dot`（向量数量积）/`slope_product`（斜率之积）/`status`（条件满足判定）/`distance_point_line`（点到直线距离公式）。

     - `renderReadouts` 改为调用 `computeReadout` 输出真实计算值（坐标格式化为 `(x, y)`，`status` 类型直接显示判定文案），不再对9种类型显示占位文字"动态量"。新增 `renderConstant` 支持 `board.constant` 字段显示"当前计算值 vs 理论定值"对照（用于斜率之积/数量积定值类题目）。
     - `drawRange` 改为优先从 `rangeBar.of` 指向的 readout 计算值取进度比例，不再直接拿滑块原始参数值（如角度0–180°）硬套弦长/面积等目标量程，修正了此前量纲不匹配导致进度条数值错误的问题。
     - CSS 补充 `.edulab-chip-highlight`（高亮显示 highlight:true 的关键readout）与 `.edulab-constant-note`（定值对照提示框）两个样式类。
